What is a Radix?

A radix is the base of the number. It is what we use to interpret a string of digits, or a number.
Normally, we use base10, or decimal, when we write numbers as strings of digits.
In other words, any string of digits you see is most likely a decimal number.

A string of digits of length n:
dn−1dn−2...d1d0 d_{n-1}d_{n-2}...d_1d_0 dn−1​dn−2​...d1​d0​

has the numeric value:
d0+d1∗10+...+dn−2∗10n−2+dn−1∗10n−1 d_0 + d_1 * 10 + ... + d_{n-2}* 10^{n-2} + d_{n-1} * 10^{n-1} d0​+d1​∗10+...+dn−2​∗10n−2+dn−1​∗10n−1
Computed in base10.

For example, the string of digits 4273 represents the value:
3+7∗10+2∗102+4∗103=3+70+200+4000 3 + 7*10 + 2*10^2 + 4*10^3= 3 + 70 + 200 + 4000 3+7∗10+2∗102+4∗103=3+70+200+4000
= 4 thousands, 2 hundreds, 7 tens and 3

It’s important to distinguish between a string of digits and its value. The value or quantity can be represented in many forms, such as the form I used above in the english language. A string of digits in base10 is one way to represent that value. Another way is roman numerals:
MMMMCCLXXIII

We can also represent it in a base other than 10, say 8.
We will learn how to do it soon, but for now, let’s try: 10261
102618=1+6∗8+2∗82+0∗83+1∗84=1+6∗8+2∗64+4096=4273 10261_8 = 1 + 6 * 8 + 2 * 8^2 + 0 * 8^3 + 1 * 8^4 = 1 + 6 * 8 + 2 * 64 + 4096 = 4273 102618​=1+6∗8+2∗82+0∗83+1∗84=1+6∗8+2∗64+4096=4273
4 thousands, 2 hundreds, 7 tens and 3

So a base is just what we use to represent a numeric value in a string of digits.
To distinguish between bases, we enclose it in parentheses, with n outside:
(1221211)3 (1221211)_3 (1221211)3​
is in base 3.

Base n digits

For any base n, we can use digits from 0 to n-1
If the base is greater than 10, we need to assign new symbols to the numeric values 10, 11, 12 etc…
Mathematicians commonly use the alphabet from A for those values.
So A corresponds to 10, B corresponds to 11, C corresponds to 12, and so on.
Example:
In base 16, the string of digits:
(10B1)16=(4273)10 (10B1)_{16} = (4273)_{10} (10B1)16​=(4273)10​
We will learn how to get it shortly, but to check, multiply each digit by the multiple of 16, as above.
1+11∗16+0∗162+1∗163=4273 1 + 11*16 + 0 * 16^2 +1*16^3 = 4273 1+11∗16+0∗162+1∗163=4273

How to write a numeric value in any base n?

Consider the numeric value represented above in decimal by the string of digits: 4273
How do we represent it in a string of digits in base 8? or for any base n?

Here’s a procedure, for N = string of digits in decimal:

  1. Divide the number by n in decimal arithmetic
  2. The remainder part is the current digit of the new string in base n, starting from the right.
  3. Repeat the procedure for the integer part of the division
  4. Stop when the result of division is 0

An example:
Writing 4273 it in base 8, or octal:

- 4273 / 8 = 534 with a remainder 1
               1 is the first digit in the result, from the right
- 534 / 8 = 66 with a remainder 6
               6 is the second digit in the result. 
- 66 / 8 = 8 with a remainder 2
               2 is the third digit in the result
- 8 / 8 = 1 with a remainder 0
               0 is the fourth digit in the result     
- 1 / 8 = 0 with a remainder 1
               1 is the fourth digit in the result

Done.
10261 is the result

Writing it in base 2, or binary

- 4273 / 2 = 2136 with a remainder 1 
               1 is the 1st digit in the result, from the right
- 2136 / 2 = 1068 with a remainder 0
               0 is the 2nd digit in the result
- 1068 / 8 = 534 with a remainder 0
               0 is the 3rd digit in the result
- 534 / 2 = 267 with a remainder 0
               0 is the 4th digit in the result     
- 267 / 2 = 133 with a remainder 1
               1 is the 5th digit in the result
- 133 / 2 = 66 with a remainder 1
               1 is the 6th digit in the result       
- 66 / 2 = 33 with a remainder 0
               0 is the 7th digit in the result     
- 33 / 2 = 16 with a remainder 1
               1 is the 8th digit in the result 
- 16 / 2 = 8 with a remainder 0
               0 is the 9th digit in the result 
- 8 / 2 = 4 with a remainder 0
               0 is the 10th digit in the result 
- 4 / 2 = 2 with a remainder 0
               0 is the 11th digit in the result
- 2 / 2 = 1 with a remainder 0
               0 is the 12th digit in the result
- 1 / 2 = 0 with a remainder 1
               1 is the 12th digit in the result

Done.
1000010110001 is the result.

Writing it in base 16, or hex:

- 4273 / 16 = 267 with a remainder 1 
               1 is the 1st digit in the result
- 267 / 16 = 16 with a remainder 11
               B is the 2nd digit in the result
- 16 / 16 = 1 with a remainder 0
               0 is the 3rd digit in the result
- 1 / 16 = 0 with a remainder 1
               1 is the 4th digit in the result

Done.
10B1 is the result.

What is the point in having different bases?

The last example shows just that; Remember that computers store information in bits, which can be either 0 or 1.
To store 4273 somewhere in the computer, it has to be converted to base2 or binary first, then stored in disks, solid state memory, cache registers, etc…

Also, in different contexts, bases other than 10 are preferred.

Converting from base n to decimal

To convert a number written in any base n to decimal, we simply expand it, multiplying each digit by the corresponding multiple of n.
We do the arithmetic in base 10.
This is basically what we have done to get the value of the number.
Example:
12212113=1+1∗3+2∗32+1∗33+2∗34+2∗35+1∗36 1221211_3 = 1 + 1*3 + 2*3^2 + 1*3^3+2*3^4+2*3^5+1*3^6 12212113​=1+1∗3+2∗32+1∗33+2∗34+2∗35+1∗36
=142610 =1426_{10} =142610​
or a thousand, four hundreds, twenty and six.

Converting fractions

Let’s do the simplest case first:

Converting from base n to decimal

Here we simply multiply each digit after the decimal point by the the reciprocal of the corresponding multiple.
Except that we start from 1 as the power, not 0, and we move left to right.

Example:
(0.122)3=1∗13+2∗132+2∗133 (0.122)_3 = 1 * \frac 13 +2 * \frac1{3^2}+ 2 * \frac1{3^3} (0.122)3​=1∗31​+2∗321​+2∗331​
=13+232+233=933+633+233=1727≈0.62962 =\frac 13 +\frac2{3^2}+ \frac2{3^3} = \frac9{3^3}+\frac6{3^3}+\frac2{3^3}=\frac{17}{27} \approx 0.62962 =31​+322​+332​=339​+336​+332​=2717​≈0.62962

Converting from decimal to base n

Multiply the fraction by n, taking the integer part in each step as the current digit, and repeating with the fraction part till we get a 0.
Note that we do arithmetic in decimal.
0.810=(x)8 0. 8_{10} = (x)_8 0.810​=(x)8​

0.8 * 8 = 6.4 => 6
0.4 * 8 =  3.2 => 3
0.2 * 8 = 1.6 => 1
0.6 * 8 = 4.8 => 4
0.8 * 8 = 6.4 => 6
0.4 * 8 = 3.2 => 3
...
x = 0.631463...

Notice that it repeats. Most fractions do when converted from decimal to other bases.
In fact, I tried to find an example that doesn’t but only found trivial ones, like 0.5 (Try to convert it from decimal to octal)
Also note that each digit needs to be converted to the destination base.
In the above example we did not need to do anything because 8 < 10, but let’s try from decimal to hex:
0.810=(x)16 0. 8_{10} = (x)_16 0.810​=(x)1​6

0.8 * 16 = 12.8 => 12 => C
0.8 * 16 = 12.8 => 12 => C
...
x = 0.CC...

Note that 12 is converted to C in hexadecimal. This will repeat too.

Converting between bases other than decimal

We learned how to convert a number written in decimal to any other base n. We also learned how to convert it back to decimal by evaluating its value.
So to convert from a base a, say 8, to base b, say 16, we can convert from 8 to 10, then from 10 to 16.
That’s fine, but there’s a faster way to directly convert.

Direct Conversion

If we have a number N written in base a:
dn−1dn−2...d1d0 d_{n-1}d_{n-2}...d_1d_0 dn−1​dn−2​...d1​d0​

How do we convert it to base b?

We can write N as:
d0+a∗(d1+...a∗(dn−1+a∗dn))) d_0 + a * (d_1 + ... a * (d_{n-1} + a * d_n))) d0​+a∗(d1​+...a∗(dn−1​+a∗dn​)))
I won’t show the proof here, but the number written in base b can be obtained if we write every digit of N, and a, in terms of the destination base b:

N=db0+ab∗(db1+...ab∗(dbn−2+ab∗dbn−1))) N= db_0 + a_{b} * (db_1 + ... a_b * (db_{n-2} + a_b * db_{n-1}))) N=db0​+ab​∗(db1​+...ab​∗(dbn−2​+ab​∗dbn−1​)))
We do the arithmetic in base b, then the result will be the number written in base b.
inin−1...i1i0 i_ni_{n-1}...i_1i_0 in​in−1​...i1​i0​
The key point here is that we need to do the arithmetic in the destination base.
Up to now, we have been doing arithmetic in decimal; we are used to that, so it was easy for us.
Now, to do arithmetic in other bases, we can use a table to make things easier for us.

Example:

7358=x7 735_8 = x_7 7358​=x7​
Convert 735 from octal to base 7

5+8∗(3+8∗7) 5 + 8 * (3 + 8 * 7) 5+8∗(3+8∗7)
In base 7:
5+11∗(3+11∗10)=5+11∗(3+110)=5+11∗113=5+1243=1251 5 + 11 * (3 + 11 * 10)\\ = 5 + 11 * (3 + 110)\\ = 5 + 11 * 113 \\=5 + 1243\\ = 1251 5+11∗(3+11∗10)=5+11∗(3+110)=5+11∗113=5+1243=1251
How did I get this result? Remember that I’m calculating in base 7.
So when I multiply 10*11, I don’t just do it in my head using normal decimal arithmetic, instead I use a procedure to multiply:
107∗117=0∗11+10∗11=0+110=1107 10 _7 * 11_7 \\= 0*11 + 10*11 = 0 + 110 = 110_7 107​∗117​=0∗11+10∗11=0+110=1107​
Then when I add that to 3, I simply add it to the rightmost digit: 113
I multiply that by 11:
117∗1137=33+110+1100=143+1100=1243 11_7 * 113_7 = 33 + 110 + 1100 = 143 + 1100 = 1243 117​∗1137​=33+110+1100=143+1100=1243
Notice how I multiply each digit alone, and I add zeros to the right for each digit after the first one.
Now when I add 5 to 1243, I don’t get 1248, because there’s no 8 in base 7; I have to carry over:
5+3=8=117 5+3 = 8 = 11_7 5+3=8=117​
I replace 3 by 1 and add 1 to 4.

Here, 2 * 7 is 14, but in octal it’s 16. We put it before the decimal place as we normally do in decimal arithmetic.

Conversion using long division

In converting integers, we used the destination radix (base). We can also convert numbers between bases directly in the source radix arithmetic, using long divison.

We divide the original number in base a by b (written in base a); the remainder is going to be the current digit, starting from the left, and we repeat using the dividend, until we reach zero.
